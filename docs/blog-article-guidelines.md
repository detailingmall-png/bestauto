# Правила написания статей блога — bestauto.ge

> Источник правды для всех новых статей из `docs/content-roadmap.md`. Перед публикацией пройти по чек-листу внизу.

## 1. Длина

**RU: 1300–2200 слов.** Единый диапазон для всех типов статей.
- Минимум 1300 — меньше не ранжируется в competitive HF-нишах
- Максимум 2200 — иначе читатель не дочитает; разбить на 2 статьи

**KA:** +10-20% к RU объёму (грузинский в среднем длиннее).
**EN:** −10-15% от RU (английский компактнее).

Длина распределяется по структуре:
- Intro: ~100-150 слов
- Основные H2-секции (5-7 штук): ~200-350 слов каждая
- FAQ (5 вопросов × 60-100 слов)
- Заключение: ~100-150 слов

## 2. Структура

Стандартный скелет статьи (обязательный порядок):

```
1. Hero cover (fullbleed фото)
2. H1 (title из roadmap)
3. Intro (80–150 слов): проблема → обещание → для кого эта статья
4. Оглавление (table of contents) — опционально, для статей 1500+ слов
5. Основные H2-секции (3–7 штук):
   - Каждая начинается с H2
   - Внутри допустимы H3 подсекции
   - 150–350 слов на H2
6. Декоративные элементы (опционально):
   - Таблица сравнения
   - Список bullet-pointов
   - Блок с примером («кейс из практики BESTAUTO»)
   - Блок с предупреждением («⚠ Избегайте: ...»)
7. Заключение (80–150 слов): итог + что делать дальше
8. FAQ-блок (3-5 вопросов) — обязательно для SEO (FAQPage schema)
9. CTA-блок с ссылкой на сервисную страницу
10. Related services (3-4 карточки сервисов)
```

**Правила заголовков:**
- **ровно один H1** на странице (совпадает с Title из roadmap)
- H2 → H3, без пропусков уровней (не H2→H4)
- H2 должен содержать primary HF или вариант (см. §3)

## 3. SEO-механика

### HF-ключ (обязательно)
- **Primary HF** из roadmap присутствует в:
  - `<title>` (meta)
  - H1
  - первые 100 слов (intro)
  - ≥1 H2-заголовке
  - meta description
  - Alt текст минимум одной картинки
- **Плотность**: 3–6 упоминаний на 1000 слов. Больше = spam, меньше = недосигналено.
- **Вариативность**: использовать 2-3 варианта HF (из словаря `seo-service-keywords.ts`):
  - Например для `/polishing`: «полировка машины» + «полировка авто» + «полировка кузова»
  - НЕ повторять 8 раз одну и ту же точную фразу

### Meta title
- 50–65 символов (под Google SERP)
- Формат: `{Primary HF} в Тбилиси — {USP} | BESTAUTO`
- НЕ дублирует Title страницы 1-в-1: title может быть длиннее, meta-title компактнее

### Meta description
- 140–160 символов
- Отличается от title — другой angle (напр. title про «что это», desc про «как получить»)
- Содержит price signal или CTA («цена от X ₾», «бесплатная консультация»)

### Schema (JSON-LD, уже автоматом через [...slug].astro)
- `Article` + `BlogPosting` — базовые
- `FAQPage` — если есть FAQ-блок (обязательно)
- `BreadcrumbList` — есть глобально

### Canonical + hreflang
- Canonical = текущая URL без query-строки
- hreflang на все 3 языка (автоматом в seo.ts — не трогать)

## 4. Internal linking (blog → service)

### Обязательно
- **2–3 ссылки** на сервисные пиллар-страницы в теле статьи
- Использовать editorial injection механизм (`/blog-links <slug>` slash-команду)
- Анкор = реальный ВЧ-запрос из `seo-service-keywords.ts`, НЕ описательная фраза

### Диверсификация анкоров (Penguin protection)
- 60% — long-tail HF («полировка кузова автомобиля»)
- 30% — mid-frequency («полировка авто»)
- 10% — generic («узнать цены»)
- Никогда не 3 × короткий exact-match

### Cross-linking между статьями
- Если статья упоминает топик другой статьи → ссылка на неё (в теле, не в footer)
- Например статья про `ceramic-coating-maintenance` должна ссылаться на `maintenance-wash-ceramic`
- Правила cross-ссылок пока НЕ автоматизированы — добавляем вручную в HTML

### Ссылки наружу
- Допустимы на источники (статьи законов Грузии, official Tilda, бренды материалов)
- `rel="nofollow noopener"` для не-партнёрских ссылок
- Минимум 2 external links на длинную статью повышают E-E-A-T

## 5. Voice & tone

Следует общему бренд-войсу из `ui-ux-guide.md` §15, плюс для блога:

### Критерий качества
Текст должен читаться как **редакторский текст профессионального журналиста** — не машинный перевод, не AI-шаблон. Главный тест: если подписать чужим именем в отраслевом журнале — пройдёт ли редактуру? Если нет — переписать.

### Принципы
- **Профессиональный, но не сухой.** Мы эксперты, но пишем для водителя в Тбилиси.
- **Говорим «вы», не «вам»** (прямое обращение).
- **Нативность языка:** KA/RU/EN должны читаться как родные, не калька друг с друга. Идиомы, ритм, связки — из целевого языка.
- **Избегаем:**
  - Воды («важно отметить», «как известно», «многие водители знают», «стоит подчеркнуть», «таким образом»)
  - Штампов («качественно, быстро, недорого»)
  - Снисходительности («конечно же», «наверняка вы понимаете»)
  - Clickbait («шокирующая правда о...», «10 секретов...»)
  - AI-клише («давайте разберёмся», «в этой статье мы рассмотрим», «это позволит вам...»)
  - Машинно-переведённых конструкций («является важным», «может быть использован»)
- **Предпочитаем:**
  - Конкретные цифры (не «долго», а «12-18 месяцев»)
  - Конкретные бренды (см. §13 — whitelist)
  - Локальные примеры («летом в Тбилиси температура в салоне достигает 55°C»)
  - Реальные прайсы со ссылкой на /prices (см. §13)
  - Честные оговорки («этот метод не сработает если...»)
  - Короткие предложения там, где можно; длинные — только для технических объяснений

### Запрещено
- Plagiarism или rewrite конкурента близко к оригиналу
- Фейковые статистики без источника
- Эмодзи в заголовках и тексте
- «Мы — лучшая студия в Тбилиси» — бренд-голос анти-хвастливый
- Выдуманные/ориентировочные/устаревшие цены (см. §13)
- Бренды, с которыми BESTAUTO не работает (см. §13)

### Оценочные слова
- Используем «один из», «часто», «обычно», «на практике»
- НЕ используем: «лучший», «самый», «№1», «единственный»

### Чистовик с первого раза
Текст пишется сразу как финальный. Нет режима «draft → правка» — каждая статья проходит критерии этого документа до публикации. Если качество ниже — переписать до того, как предъявить.

## 6. Мультиязычность (KA / RU / EN)

- **Все 3 языка синхронно одним PR** — не копим языковые долги
- **Порядок написания:** сначала RU (как draft-prompt) → KA → EN. RU даёт смысловую рамку, KA и EN — переложение, не дословный перевод
- **Не использовать машинный перевод** для title, hero subtitle, CTA. FAQ и тело допустим машинный + ручная правка
- **KA-специфика:**
  - Можно включать Russism-дубли в мета/FAQ/schema (`პალიროვკა`, `ლაბავოი`) — но НЕ в visible hero/H1/H2 (ui bolted gone April 2026)
  - Georgian длиннее RU на 10-20% — проверять переносы на mobile 375px
  - Не использовать `მანქანის ფირი` для vinyl (зарезервировано за PPF в словаре)
- **RU-специфика:**
  - Geo: добавлять «в Тбилиси» / «в Грузии» где уместно (но не forced)
  - Избегать «штучных» русизмов («разбираем», «гоним», «заморачиваться»)
- **EN-специфика:**
  - Всегда добавлять «Tbilisi, Georgia» — иначе US-noise
  - Use British English when mentioning colour (uniform with existing articles — `colour`)

## 7. Медиа (фото, видео)

**Текущая фаза: без фото.** Пишем только текстовые статьи. Фотографии будут подбираться и добавляться отдельной партией позже — у BESTAUTO есть архив реальных работ, но их нужно тщательно отобрать под каждую статью, это отдельный процесс.

Что НЕ делаем сейчас:
- Не вставляем stock-фото
- Не вставляем AI-generated картинки
- Не оставляем placeholder-изображения

Что делаем:
- Текст структурируется так, чтобы **работать без визуалов** (больше подзаголовков, таблиц, списков — они дают визуальную разбивку без фото)
- В коде статьи можно оставить HTML-комментарий `<!-- TODO: photo of polishing process -->` где фото логично — это поможет фото-редактору потом
- После добавления фото partии — отдельный PR с обновлением медиа

**Будущие правила** (для справки, когда дойдём до медиа):
- WebP, 1920×1080 hero / 800-1200 inline
- Alt с HF-вариантом
- Только реальные работы BESTAUTO (не stock)
- Before/after — 2-колоночная таблица или слайдер

## 8. Локальная специфика

### Geo-qualifiers
- «в Тбилиси» — по умолчанию для всех статей
- «в Грузии» — для законодательных (`tint-60-percent-legal-georgia`) и широких
- «Сабуртало / Глдани» — если есть контекст геолокации (напр. в FAQ), но НЕ в title/H1/meta

### Climate / seasonality
- Тбилиси-специфика: 55°C летом, пыль, соль зимой, горные поездки
- Используем это в статьях про защиту/уход

### Законодательство
- Только свежие данные (Georgia tinting law 2024, не устаревшие)
- Ссылка на официальный источник (gov.ge, автомотив регуляторы)

### 8a.2. Мягкая рекомендация PPF в статьях про полировку/керамику/общие

**Идея:** PPF (защитная плёнка) — наша главная маржинальная услуга и реально самое надёжное решение для долгосрочной защиты кузова. В статьях смежных категорий мягко подводим читателя к этой мысли.

**Где применять:**
- Все статьи кластера `/polishing`
- Все статьи кластера `/ceramiccoating`
- Общие детейлинг-статьи (`/carwash` где уместно, `GEN`)

**Как:**
- В секции «когда чего не хватает» или «что делать для максимальной защиты» — аккуратно упомянуть PPF как следующий уровень
- Использовать реальные сравнения: полировка возвращает вид на короткий срок, керамика облегчает уход, **PPF** — единственный физический барьер от сколов и камней
- Добавлять 1 internal link на `/ppf-shield-wrapping` через editorial injection (это уже встроено в наш механизм)
- Не навязывать: если статья про DIY-полировку — уместно одно упоминание в заключении, не 5 по тексту

**Как НЕ делать:**
- Не превращать каждую статью в PPF-рекламу
- Не принижать другие сервисы («полировка — бесполезна»)
- Не обещать то, чего PPF не делает («полная защита от любых повреждений»)

**Примерная формулировка:**
> «Полировка возвращает блеск, но не защищает кузов от новых сколов. Если важна долгосрочная защита — более надёжное решение [защитная плёнка PPF](/ppf-shield-wrapping), которая принимает удар на себя.»

**Где НЕ применять:**
- Статьи кластера `/ppf-shield-wrapping` (там PPF и так основной топик)
- Статьи про тонировку, химчистку, ремонт стекла — там PPF нерелевантен

### 8a.3. Whitelist брендов по кластерам

В статьях по каждому кластеру упоминаем **только** бренды, с которыми работает BESTAUTO. Отклонение = критическая ошибка: читатель придёт с конкурентским запросом, которого у нас нет.

| Кластер | Whitelist-бренды |
|---|---|
| **/ppf-shield-wrapping** (PPF) | **Llumar**, **LuxArmor**, **Quantum** |
| **/ceramiccoating** (керамика) | **Gyeon** |
| **/auto-glass-tinting** (тонировка) | **LLumar**, **LuxArmor** (только эти два) |
| **/carwash** (химия для мойки) | **Koch Chemie** |
| **/polishing, /vinyl-wrapping, /windshield-repair, /interior-cleaning** | **НЕТ whitelist → НЕ упоминать конкретные бренды вообще** |

**Ключевое правило (2026-04-17 update):** если для кластера whitelist не определён — использовать **дженерик-формулировки** («премиум-винил», «европейские плёнки», «профессиональная автохимия», «инжектор смолы»), НЕ называть конкретные бренды. Это относится и к сравнительным таблицам.

**Что НЕ делать:**
- PPF: не упоминать XPEL, Stek, 3M, SunTek в рекомендательном контексте
- Керамика: не упоминать Gtechniq, Ceramic Pro, Nanolex, CarPro как «наши» — только Gyeon
- Тонировка: не упоминать 3M, SunTek, Johnson, Solar Gard — только LLumar и LuxArmor
- Мойка: не называть SONAX, Meguiar's, Chemical Guys как «нашу» химию — только Koch Chemie
- **Винил: не упоминать Hexis, KPMF, Oracal, Avery Dennison, Ravoony, Teckwrap, 3M ни в каком контексте**
- **Ремонт стекла: не упоминать 3M, Equalizer, GlassGlue и прочие индустриальные бренды материалов**
- **Полировка: не упоминать Meguiar's, SONAX, Chemical Guys, Menzerna, Turtle Wax, Mothers и др.**
- **Химчистка: кроме Koch Chemie (и то аккуратно в мойке — для интерьера generic)** — не упоминать Tornador, Leather Master, ValetPRO и др.
- Ни в одном кластере не ставить whitelist-бренд в один список с чужими брендами без выделения «с чем работает BESTAUTO»

**Сравнительные таблицы (2026-04-18 update — usage's directive):**
- НЕЛЬЗЯ упоминать любые бренды, не входящие в whitelist BESTAUTO, ни в одном контексте, включая сравнительные таблицы.
- Сравнения — только между брендами whitelist (например, Llumar vs LuxArmor vs Quantum) или через дженерики («премиум-сегмент», «бюджетный сегмент», «европейские производители», «китайские производители», «международные премиум-производители»).
- Запрещено: XPEL, Stek, SunTek, 3M, KDX, Hexis, KPMF, Oracal, Avery Dennison, Ravoony, Teckwrap, Gtechniq, Ceramic Pro, Nanolex, CarPro, Meguiar, SONAX, Chemical Guys, Turtle Wax, Mothers, Menzerna, Solar Gard, Johnson, Lintec, Huper Optik, CPFilms, Tornador, Leather Master, ValetPRO, Cquartz, Nasiol, Opti-Coat — и любой другой бренд, с которым BESTAUTO не работает.
- Это касается и упоминаний в meta description, frontmatter, sources, body, FAQ, заключении.

**Формулировки для статей:**

PPF-статьи:
> «BESTAUTO работает с тремя премиум-брендами PPF: Llumar, LuxArmor и Quantum. Каждый имеет 10-летнюю гарантию производителя.»

Ceramic-статьи:
> «В студии используется керамика Gyeon — корейский бренд с хорошей репутацией в Европе и прозрачной системой классификации по твёрдости (9H и выше).»

Tinting-статьи:
> «Для тонировки используем только LLumar и LuxArmor — два премиум-бренда с гарантией и нормальной цветостойкостью (не выцветают до фиолетового за 2 года, как дешёвые альтернативы).»

Carwash-статьи:

### 8a.4. Структура — чёткая и логичная

**Каждая статья должна иметь:**
- Логический скелет: проблема → варианты решения → рекомендация → FAQ
- Навигацию: оглавление для статей 1500+ слов (обязательно)
- Визуальную разбивку: H2 каждые 200-350 слов, короткие абзацы (3-5 строк на mobile)
- Списки и таблицы там, где информация структурированная (сравнения, этапы, цены)
- Итоговый блок с ключевыми выводами в конце (bullet-list из 3-5 пунктов)

**Проверка чёткости структуры:**
- Если выкинуть все H2 и прочитать только их список — пойдёт ли логика?
- Может ли читатель найти ответ на конкретный вопрос за 10 секунд?
- Нет ли повторов между H2-секциями?

## FAQ по написанию

### Длина попадает между 1300 и 2200 — как выбрать точное число?
Смотри primary интент:
- Пользователь ищет конкретный ответ («как починить скол») → ближе к 1300-1500
- Пользователь сравнивает варианты перед покупкой → ближе к 1800-2200
- Чем больше технической глубины — тем длиннее; не добавлять воды для объёма

### Что делать если HF-ключ не в словаре?
1. Проверить GSC — есть ли impressions
2. Если да — добавить в `seo-service-keywords.ts` с комментарием-источником (GSC: X imp / Y clicks)
3. Если нет — не использовать; взять из словаря

### Можно ли «мы — лучшая студия в Тбилиси»?
Нет. Бренд анти-хвастливый (см. §5). Лучше: «одна из немногих студий в Тбилиси с гарантией 10 лет на PPF».

### Что если актуальная цена изменилась после снимка в §8a.1?
Проверить https://bestauto.ge/prices. Если цифра отличается — использовать актуальную (сайт = источник правды). Снимок в §8a.1 может устаревать; сверять раз в месяц.

### Сколько раз в статье про полировку упоминать PPF?
Один раз. Обычно в заключении или в секции «когда полировки недостаточно». Link → /ppf-shield-wrapping. Если упомянуть 2-3 раза — получится реклама, качество статьи страдает.

### Как упомянуть PPF если статья совсем не про защиту (напр. про мойку)?
Не навязывать. Если статья про мойку, PPF может упоминаться только если это естественно (напр. «при мойке машины с PPF — ...»). Не делать специальных секций про PPF там, где это не релевантно.

### Можно упоминать XPEL или 3M в PPF-статьях?
Только в сравнительных таблицах (и с аккуратной подписью «мы работаем с Llumar/LuxArmor/Quantum»). Отдельные рекомендации этих брендов — нет.
