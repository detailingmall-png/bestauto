# Design System for bestauto.ge

> ОБЯЗАТЕЛЬНАЯ ИНСТРУКЦИЯ. Перед любым изменением на сайте (редактирование блока, добавление нового блока, создание страницы, изменение стилей) — прочитай и следуй этим правилам.
> Основа: доказательное исследование ~/ui-ux-research.md (NNGroup, Baymard, WebAIM, Google Research, WCAG 2.1) + мировые практики Material Design, Carbon, Polaris, Atlassian.

---

## 0. Принципы дизайна и бренд-идентичность

Пять принципов, которыми руководствуется каждое визуальное решение на bestauto.ge:

### 1. Премиальная сдержанность
Тёмный фон, золотой акцент, минимум цветов. Роскошь = пустота и чистота, НЕ пестрота. Если сомневаешься — убери элемент, а не добавляй.

### 2. Доверие через прозрачность
Цены видны на каждой странице. Процесс работы объяснён по шагам. Отзывы — настоящие (Google Reviews API). Никогда не скрывай информацию от клиента.

### 3. Ориентация на действие
Каждый экран ведёт к записи на осмотр. CTA описывает РЕЗУЛЬТАТ ("Записаться на бесплатный осмотр"), а не действие ("Отправить"). Один основной CTA на секцию.

### 4. Mobile-first для Грузии
Три языка: грузинский (основной), русский, английский. Touch-first интерфейс. Все решения проверяются на viewport 375px. Грузинский текст длиннее — тестировать всегда.

### 5. Скорость как фича
Astro SSG, статический HTML, lazy loading, WebP, один шрифт (TildaSans). Никаких тяжёлых зависимостей. Быстрый сайт = доверие.

### Tilda-блоки: стандартизация типографики (ОБЯЗАТЕЛЬНО)

Tilda-экспортированные блоки (t502, t508, t585 и др.) содержат inline-стили `data-customstyle="yes"` с произвольными font-size. Эти значения НЕ ЯВЛЯЮТСЯ стандартом — они были заданы вручную в редакторе Tilda и часто различаются между страницами.

**CSS-переопределения** в `bestauto-custom.css` стандартизируют размеры с помощью `!important`:

| Tilda-блок | Элемент | CSS-селектор | Desktop | Tablet | Mobile |
|------------|---------|-------------|---------|--------|--------|
| **t205** (hero cover) | H1 заголовок | `.t182__title [data-customstyle]` | 48px | 40px | 32px |
| **t33** (секция t017) | H2 заголовок | `.t017__title [data-customstyle]` | 36px | 32px | 28px |
| **t255** (секция t225) | H2 заголовок | `.t225__title [data-customstyle]` | 36px | 32px | 28px |
| **t502** (нумерованные преимущества) | Заголовок секции | `.t502 .t-section__title` | 36px | 32px | 28px |
| | Заголовок пункта | `.t502__title` | 22px | 20px | 18px |
| | Описание | `.t502__blockdescr` | 15px | 14px | 14px |
| **t508** (список с иконками) | Заголовок секции | `.t508 .t-section__title` | 36px | 32px | 28px |
| | Заголовок пункта | `.t508 .t-name` | 22px | 20px | 18px |
| | Описание | `.t508__descr` | 15px | 14px | 14px |

**Правила:**
- Inline `data-customstyle` с `font-size` в t502/t508 — игнорируется CSS через `inherit !important`.
- При добавлении новых блоков в Tilda: НЕ задавать customstyle font-size. Размеры берутся из `bestauto-custom.css`.
- Если блок выглядит иначе на другой странице — проблема в inline customstyle. Проверь `bestauto-custom.css`.

### Межстрочный интервал (line-height)

| Элемент | Значение |
|---------|----------|
| Заголовки (1-2 строки) | 1.2-1.35 |
| Основной текст | 1.5-1.6 |
| Мелкий текст | 1.4-1.5 |

### Длина строки

- Десктоп: 50-75 символов (оптимум: 66).
- Мобильный: 30-50 символов.
- Используй `max-width` на текстовых блоках для ограничения.

### Выравнивание

- **Левое выравнивание** — стандарт для текста длиннее 2 строк.
- Центральное выравнивание допустимо ТОЛЬКО для: заголовков секций (1-2 строки), кнопок, коротких подзаголовков.
- **ЗАПРЕЩЕНО**: `text-align: justify` — нарушает доступность (WCAG).

## 5. Spacing System (8px grid)

> Источники: Material Design 3, IBM Carbon, Atlassian, Shopify Polaris, NNGroup, EightShapes (Nathan Curtis).

### Базовая шкала (ОБЯЗАТЕЛЬНО)

```
4 | 8 | 12 | 16 | 24 | 32 | 40 | 48 | 64 | 80 | 96 | 160
```

Кастомные `ba-*` компоненты используют 8px grid. Tilda-блоки используют 15px increments (`0, 30, 45, 60, 75, 90, 105, 135, 150`). Не смешивай системы внутри одного компонента.

### Hero Section Spacing

```
DESKTOP (>980px)                     MOBILE (<=640px)
+----------------------------------+ +--------------------+
|        64px top padding          | |   32px top pad     |
|                                  | |                    |
|   H1 (42px font)                | |  H1 (28-30px font) |
|                                  | |                    |
|        24px gap                  | |    16px gap        |
|                                  | |                    |
|   Subtitle (18px font)          | |  Subtitle (15-16px)|
|                                  | |                    |
|        32-40px gap               | |    24-32px gap     |
|                                  | |                    |
|   [ CTA Button 48px+ tall ]     | |  [CTA 48px+ tall]  |
|                                  | |                    |
|       96px bottom padding       | |   64px bottom pad  |
+----------------------------------+ +--------------------+
```

| Связь элементов | Desktop (>980px) | Tablet (641-980px) | Mobile (<=640px) |
|-----------------|-----------------|-------------------|-----------------|
| H1 → Subtitle | **24px** | **20px** | **16px** |
| Subtitle → CTA | **40px** | **32px** | **24px** |
| Hero padding top | **64px** | **48px** | **32px** |
| Hero padding bottom | **96px** | **96px** | **64px** |

**Принцип Proximity (NNGroup):** Subtitle-CTA gap ВСЕГДА больше H1-Subtitle gap. Заголовок и описание — одна группа, CTA визуально отделена.

### Responsive: data-field-top значения для Zero Block hero

Tilda Zero Block использует абсолютное позиционирование с `axisy="center"` (top — смещение от центра артборда).

## 8. Формы

### Количество полей

- **Максимум 3-4 поля** для формы записи (услуга, телефон, модель авто).
- Каждое лишнее поле = минус 3-5% конверсии (Baymard).
- НЕ добавлять поля без крайней необходимости.

### Структура поля (ОБЯЗАТЕЛЬНО)

```html
<div class="ba-field">
  <label for="ba-phone-STUDIO_ID">ВАШ ТЕЛЕФОН *</label>
  <input
    type="tel"
    id="ba-phone-STUDIO_ID"
    placeholder="+995 5XX XXX XXX"
    required
    aria-required="true"
    autocomplete="tel"
  >
  <div class="ba-error" id="ba-phone-error-STUDIO_ID" role="alert">
    Сообщение об ошибке
  </div>
</div>
```

**ОБЯЗАТЕЛЬНО**:
- `id` на каждом input, `for` на каждом label — они ДОЛЖНЫ совпадать.
- `required` или `aria-required="true"` на обязательных полях.
- `role="alert"` на сообщениях об ошибках.
- `aria-describedby` если есть подсказка.
- `autocomplete` атрибут (tel, name, email и т.д.).

### Размеры полей

| Свойство | Значение |
|----------|---------|
| Высота input/select | минимум 48px |
| Font-size input | 14-16px |
| Font-size label | 13-14px |
| Padding input | 12px 14px |
| Border-radius | `var(--ba-radius-md)` (8px) |

### Валидация

- Инлайн-валидация (по мере ввода) — НЕ только при отправке.
- Цвет ошибки: `var(--ba-color-error)` (#ff4444) с контрастом 4.5:1.
- Текст ошибки рядом с полем, а не в alert.

```css
@media (prefers-reduced-motion: reduce) {

}
```

Это правило ДОЛЖНО быть в `bestauto-custom.css`. Оно отключает все анимации для пользователей с вестибулярными нарушениями.

## 16. Доступность (a11y) — ОБЯЗАТЕЛЬНО

### Семантика HTML

- **Один H1** на страницу. Всегда. Без исключений.
- Иерархия заголовков: H1 > H2 > H3. НЕ пропускать уровни.
- Используй семантические теги: `<nav>`, `<main>`, `<header>`, `<footer>`, `<section>`, `<article>`.
- `<button>` для действий, `<a>` для навигации. НЕ наоборот.

### Фокус

- **НИКОГДА** не удалять `outline` без замены.
- Стилизовать focus-visible:
```css
:focus-visible {
  outline: 2px solid var(--ba-color-accent);
  outline-offset: 2px;
}
```
- Все интерактивные элементы должны быть доступны с клавиатуры (Tab).
- Порядок фокуса = визуальный порядок.

### Ссылки и кнопки

- Каждая ссылка ДОЛЖНА иметь текст или `aria-label`.
- Пустые ссылки (только иконки) — обязательно `aria-label="Описание"`.
- Каждая кнопка ДОЛЖНА иметь текст или `aria-label`.

### ARIA

- Семантический HTML в приоритете. ARIA только когда HTML недостаточно.
- `aria-expanded` на кнопках, управляющих раскрытием (бургер, аккордеон).
- `aria-current="page"` на активной ссылке навигации.
- `role="alert"` на сообщениях об ошибках формы.
- НЕ злоупотреблять ARIA — неправильное ARIA хуже его отсутствия.

### Язык

- `<html lang="ru">` для /ru/, `lang="ka"` для /, `lang="en"` для /en/.
- Уже реализовано в [...slug].astro — НЕ ломать.

- Цены на видном месте (прозрачность снижает тревогу).
