# UI/UX Guide for bestauto.ge

> ОБЯЗАТЕЛЬНАЯ ИНСТРУКЦИЯ. Перед любым изменением на сайте (редактирование блока, добавление нового блока, создание страницы, изменение стилей) — прочитай и следуй этим правилам.
> Основа: доказательное исследование ~/ui-ux-research.md (NNGroup, Baymard, WebAIM, Google Research, WCAG 2.1).

---

## 1. Типографика

### Размеры шрифтов (ОБЯЗАТЕЛЬНО)

| Элемент | Десктоп (>980px) | Мобильный (<=640px) |
|---------|-----------------|---------------------|
| H1 | 36-48px | 28-32px |
| H2 | 28-36px | 22-28px |
| H3 | 20-24px | 18-20px |
| Основной текст | 16-20px | 14-16px |
| Лейблы форм | 13-14px | 12-13px |
| Мелкий текст (caption) | 12-14px | 12px |

**ЗАПРЕЩЕНО**: текст менее 12px где-либо на сайте.

### Межстрочный интервал (line-height)

| Элемент | Значение |
|---------|----------|
| Заголовки (1-2 строки) | 1.2-1.35 |
| Основной текст | 1.5-1.6 |
| Мелкий текст | 1.4-1.5 |

### Длина строки

- Десктоп: 50-75 символов (оптимум: 66).
- Мобильный: 30-50 символов.
- Используй `max-width` на текстовых блоках для ограничения.

### Выравнивание

- **Левое выравнивание** — стандарт для текста длиннее 2 строк.
- Центральное выравнивание допустимо ТОЛЬКО для: заголовков секций (1-2 строки), кнопок, коротких подзаголовков.
- **ЗАПРЕЩЕНО**: `text-align: justify` — нарушает доступность (WCAG).

### Шрифт

- TildaSans — единственный шрифт на сайте. НЕ добавлять другие.

## 2. Цвет и контраст

### Фирменная палитра

| Назначение | Цвет | Hex |
|-----------|------|-----|
| Фон основной | Чёрный | #000000 |
| Текст основной | Белый | #FFFFFF |
| Accent / CTA | Золотой | #e4c97e |
| CTA текст | Чёрный на золотом | #000000 |
| Ошибки | Красный | #ff4444 |
| Успех / WhatsApp | Зелёный | #25D366 |

### Контраст (ОБЯЗАТЕЛЬНО)

| Элемент | Минимальный контраст |
|---------|---------------------|
| Обычный текст (<24px) | 4.5:1 (WCAG AA) |
| Крупный текст (>=24px или >=18.66px bold) | 3:1 |
| CTA-кнопки (текст к фону кнопки) | 6:1-8:1 (оптимум) |
| Интерактивные состояния (hover, focus) | 3:1 |
| Placeholder-текст | 4.5:1 к фону поля |
| Границы полей ввода | 3:1 к фону |

**Проверяй контраст** перед каждым коммитом: https://webaim.org/resources/contrastchecker/

### Правила

- НЕ используй серый текст на чёрном фоне с контрастом ниже 4.5:1.
- Placeholder-ы полей формы: rgba(255,255,255,0.5) минимум (НЕ 0.04).
- Все hover/focus/active состояния должны сохранять контраст.

## 3. Навигация

### Десктоп (>980px)

- **Видимое горизонтальное меню** — НЕ бургер. Пункты: Услуги | Цены | Контакты | Блог | [Записаться] (CTA-кнопка).
- Максимум 5-7 видимых пунктов верхнего уровня (закон Хика).
- `aria-current="page"` на активной ссылке.
- Фиксированная шапка (sticky header).

### Мобильный (<=980px)

- Бургер-меню допустим ТОЛЬКО на мобильном.
- Touch target бургера: **минимум 48x48px** (можно padding вокруг иконки 28px).
- `aria-label="Меню навигации"` на бургере.
- `aria-expanded="true/false"` при открытии/закрытии.

### Хлебные крошки

- Добавлять на ВСЕ страницы кроме главной.
- Формат: Главная > Раздел > Страница.
- Размещение: сразу под header, рядом с заголовком H1.
- Используй BreadcrumbList schema (уже есть в seo.ts).

### Skip-ссылка

- Присутствует в TildaPageLayout.astro: `<a href="#main-content" class="skip-link">`.
- НЕ удалять.

## 4. Кнопки и CTA

### Размеры (ОБЯЗАТЕЛЬНО)

| Платформа | Минимальная высота | Минимальная ширина |
|-----------|-------------------|-------------------|
| Десктоп | 40px | 120px |
| Мобильный | 48px | 48px (иконка) / 120px (текст) |

### Touch targets

- **Минимум 48x48px** для ВСЕХ кликабельных элементов на мобильном.
- Расстояние между соседними touch targets: минимум 8px.
- Можно сохранить визуально компактный элемент (24px иконка), расширив область касания padding-ом.

### Стилизация CTA

```css
/* Первичный CTA */
background-color: #e4c97e;
color: #000000;
border-radius: 20px;
font-weight: 600;
padding: 12px 32px;
min-height: 48px; /* мобильный */

/* Hover */
background-color: #d4b96e; /* чуть темнее */
/* ИЛИ */
background-color: #f0d890; /* чуть светлее */
```

### Текст кнопок

- Описывай РЕЗУЛЬТАТ, а не действие: "Записаться на осмотр" (не "Отправить").
- Один основной CTA на hero-секцию. Максимум два.
- Sticky CTA на мобильном — "Записаться на бесплатный осмотр" в нижней части экрана.

## 5. Формы

### Количество полей

- **Максимум 3-4 поля** для формы записи (услуга, телефон, модель авто).
- Каждое лишнее поле = минус 3-5% конверсии (Baymard).
- НЕ добавлять поля без крайней необходимости.

### Структура поля (ОБЯЗАТЕЛЬНО)

```html
<div class="ba-field">
  <label for="ba-phone-STUDIO_ID">ВАШ ТЕЛЕФОН *</label>
  <input
    type="tel"
    id="ba-phone-STUDIO_ID"
    placeholder="+995 5XX XXX XXX"
    required
    aria-required="true"
    autocomplete="tel"
  >
  <div class="ba-error" id="ba-phone-error-STUDIO_ID" role="alert">
    Сообщение об ошибке
  </div>
</div>
```

**ОБЯЗАТЕЛЬНО**:
- `id` на каждом input, `for` на каждом label — они ДОЛЖНЫ совпадать.
- `required` или `aria-required="true"` на обязательных полях.
- `role="alert"` на сообщениях об ошибках.
- `aria-describedby` если есть подсказка.
- `autocomplete` атрибут (tel, name, email и т.д.).

### Размеры полей

| Свойство | Значение |
|----------|---------|
| Высота input/select | минимум 48px |
| Font-size input | 14-16px |
| Font-size label | 13-14px |
| Padding input | 12px 14px |
| Border-radius | 8px |

### Валидация

- Инлайн-валидация (по мере ввода) — НЕ только при отправке.
- Цвет ошибки: #ff4444 с контрастом 4.5:1.
- Текст ошибки рядом с полем, а не в alert.

## 6. Мобильный UX

### Thumb Zones

- CTA и ключевые действия — в **нижней трети экрана**.
- WhatsApp кнопка: нижний правый угол, 56x56px, position: fixed.
- Sticky CTA: нижняя часть экрана, полная ширина.

### Текст на мобильном

- **НИКОГДА** не допускать обрезание текста за краем экрана.
- Всегда проверять на viewport 375px (iPhone SE/mini).
- Добавлять `overflow-wrap: break-word` на длинные заголовки.
- Проверять `word-break` для длинных слов (грузинский язык).

### Click-to-call

- Телефонные номера ВСЕГДА оборачивать в `<a href="tel:+995550000299">`.
- На мобильном это обязательная конверсионная точка.

### Responsive breakpoints

| Breakpoint | Устройство |
|-----------|-----------|
| <=640px | Мобильный (приоритет!) |
| 641-980px | Планшет |
| >980px | Десктоп |

## 7. Изображения

### Alt-тексты (ОБЯЗАТЕЛЬНО)

- **ВСЕ** содержательные изображения ДОЛЖНЫ иметь alt с описанием: "Полировка кузова BMW в студии BESTAUTO".
- Декоративные изображения: `alt=""` + `aria-hidden="true"`.
- **НЕ** оставлять img без alt атрибута вообще.

### Формат и оптимизация

- WebP — основной формат.
- `loading="lazy"` на все изображения НИЖЕ сгиба.
- `loading="eager"` только на hero-изображение и LCP-элемент.
- `decoding="async"` на галерейных изображениях.
- Указывать `width` и `height` для предотвращения CLS.

## 8. Доверие и социальное доказательство

### Обязательные элементы

- Google отзывы — виджет на каждой странице.
- Рейтинг Google Maps — ссылка на профиль.
- Адреса + часы работы — в контактном блоке.
- Цены "от X лари" — на карточках услуг.

### Правила размещения

- 3-5 элементов доверия на секцию максимум. Больше вызывает скептицизм.
- Отзывы и рейтинг — рядом с CTA и ценами (в точках принятия решений).
- НЕ показывать trust signals слишком рано — это поднимает вопросы.

## 9. Доступность (a11y) — ОБЯЗАТЕЛЬНО

### Семантика HTML

- **Один H1** на страницу. Всегда. Без исключений.
- Иерархия заголовков: H1 > H2 > H3. НЕ пропускать уровни.
- Используй семантические теги: `<nav>`, `<main>`, `<header>`, `<footer>`, `<section>`, `<article>`.
- `<button>` для действий, `<a>` для навигации. НЕ наоборот.

### Фокус

- **НИКОГДА** не удалять `outline` без замены.
- Стилизовать focus-visible:
```css
:focus-visible {
  outline: 2px solid #e4c97e;
  outline-offset: 2px;
}
```
- Все интерактивные элементы должны быть доступны с клавиатуры (Tab).
- Порядок фокуса = визуальный порядок.

### Ссылки и кнопки

- Каждая ссылка ДОЛЖНА иметь текст или `aria-label`.
- Пустые ссылки (только иконки) — обязательно `aria-label="Описание"`.
- Каждая кнопка ДОЛЖНА иметь текст или `aria-label`.

### ARIA

- Семантический HTML в приоритете. ARIA только когда HTML недостаточно.
- `aria-expanded` на кнопках, управляющих раскрытием (бургер, аккордеон).
- `aria-current="page"` на активной ссылке навигации.
- `role="alert"` на сообщениях об ошибках формы.
- НЕ злоупотреблять ARIA — неправильное ARIA хуже его отсутствия.

### Язык

- `<html lang="ru">` для /ru/, `lang="ka"` для /, `lang="en"` для /en/.
- Уже реализовано в [...slug].astro — НЕ ломать.

## 10. Производительность

### Core Web Vitals — целевые значения

| Метрика | Цель |
|---------|------|
| LCP | <= 2.5 сек |
| INP | <= 200 мс |
| CLS | <= 0.1 |

### Правила

- **НЕ** добавлять синхронные скрипты в `<head>`.
- Все новые скрипты: `defer` или `async`.
- Новые CSS-файлы: через preload + onload если некритические.

- Шрифты: только TildaSans. НЕ подключать Google Fonts или другие.

## 11. Spacing System (8px grid)

> Источники: Material Design 3, IBM Carbon, Atlassian, Shopify Polaris, NNGroup, EightShapes (Nathan Curtis), Red Hat Design System.

### Базовая шкала (ОБЯЗАТЕЛЬНО)

Все отступы кратны 8px (4px для fine-tuning):

```
4 | 8 | 12 | 16 | 24 | 32 | 40 | 48 | 64 | 80 | 96 | 160
```

### Hero Section Spacing

```
DESKTOP (>980px)                     MOBILE (<=640px)
+----------------------------------+ +--------------------+
|        64px top padding          | |   32px top pad     |
|                                  | |                    |
|   H1 (42px font)                | |  H1 (28-30px font) |
|                                  | |                    |
|        24px gap                  | |    16px gap        |
|                                  | |                    |
|   Subtitle (18px font)          | |  Subtitle (15-16px)|
|                                  | |                    |
|        32-40px gap               | |    24-32px gap     |
|                                  | |                    |
|   [ CTA Button 48px+ tall ]     | |  [CTA 48px+ tall]  |
|                                  | |                    |
|       96px bottom padding       | |   64px bottom pad  |
+----------------------------------+ +--------------------+
```

| Связь элементов | Desktop (>980px) | Tablet (641-980px) | Mobile (<=640px) |
|-----------------|-----------------|-------------------|-----------------|
| H1 → Subtitle | **24px** | **20px** | **16px** |
| Subtitle → CTA | **40px** | **32px** | **24px** |
| Hero padding top | **64px** | **48px** | **32px** |
| Hero padding bottom | **96px** | **96px** | **64px** |

**Принцип Proximity (NNGroup):** Subtitle-CTA gap ВСЕГДА больше H1-Subtitle gap. Заголовок и описание — одна группа, CTA визуально отделена.

### Секционные отступы

| Связь | Desktop | Tablet | Mobile |
|-------|---------|--------|--------|
| Между секциями | **64-96px** | **48-64px** | **32-48px** |
| Заголовок секции → текст | **24px** | **20px** | **16px** |
| Текст → текст (абзацы) | **16-24px** | **16px** | **16px** |
| Текст → CTA кнопка | **32-40px** | **32px** | **24px** |
| Боковые отступы контента | **16-60px** | **16-32px** | **16-20px** |

### Responsive: data-field-top значения для Zero Block hero

Tilda Zero Block использует абсолютное позиционирование с `axisy="center"` (top — смещение от центра артборда).

Стандартные значения для hero элементов на bestauto.ge:

| Элемент | default | 960 | 640 | 480 | 320 |
|---------|---------|-----|-----|-----|-----|
| **H1** top | -80 | -50 | -30 | -10 | 5 |
| **H1** font | 42 | 36 | 30 | 28 | 24 |
| **Subtitle** top | 25 | 55 | 70 | 80 | 75 |
| **Subtitle** font | 18 | 18 | 16 | 15 | 14 |
| **CTA** top | 130 | 170 | 230 | 200 | 185 |
| **CTA** font | 16 | 16 | 16 | 15 | 14 |

Расстояния H1→Subtitle: 105/105/100/90/70px.
Расстояния Subtitle→CTA: 105/115/160/120/110px.
Subtitle→CTA всегда >= H1→Subtitle (принцип Proximity).

## 12. Структура страницы услуги

### Обязательные секции (в порядке появления)

1. **Hero**: H1 + краткое описание + 1 CTA "Записаться" + фото/видео
2. **Преимущества/Что даёт**: 3-5 пунктов с иконками
3. **Подробное описание**: текст + фото процесса
4. **Цены**: таблица или карточки "от X лари"
5. **Галерея**: фото работ (до/после если применимо)
6. **Отзывы**: виджет Google Reviews
7. **Контакты**: выбор студии + форма записи
8. **Связанные услуги**: карточки 3-4 других услуг
9. **Footer**: навигация + копирайт

### Правила для новых страниц

- Копировать структуру существующей страницы услуги.
- Один H1. Заголовки по иерархии H1>H2>H3.
- Цены на видном месте (прозрачность снижает тревогу).
- CTA "Записаться" — минимум в hero и в контактах.

## 12. Мультиязычность

- 3 языка: KA (основной), RU, EN.
- Все новые блоки/страницы — сразу на трёх языках.
- hreflang и canonical — автоматически через seo.ts.
- Проверять длину текста: грузинский и русский часто длиннее английского.
